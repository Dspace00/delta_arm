# UAV Control Architectures: PX4 Normalization vs SE3 Physical Control

在构建和调试我们的无人机串级控制器时，理解不同的控制架构流派至关重要。目前主要存在两种极具代表性的架构：
1. **PX4 归一化架构**：面向工程与实机的鲁棒性设计，所有控制量都在 `[-1, 1]` 之间流动。
2. **SE3 物理量纲架构**（如 `Quadrotor_SE3_Control`）：面向学术与严格数学推导，直接在 SO(3)/SE(3) 流形上进行基于物理单位（N, Nm）的控制。

下面我们将对比这两种架构，并讨论我们项目（基于 MuJoCo 仿真）该何去何从。

---

## 1. PX4 归一化架构 (Normalization Strategy)

### 1.1 为什么需要归一化？
在物理世界中，不同的无人机有不同的电机（最大推力不同）、不同的机架（力臂长度不同）和不同的转动惯量。如果控制器直接输出 $Nm$，那么一套调好的 PID 参数换个电机就完全失效了。
**PX4 的解决思路：**
将控制器输出与底层执行机构解耦。控制器只输出 **“我需要用到飞机多少比例的最大能力”**，即输出范围域被强制压缩在 `[-1.0, 1.0]`（力矩）和 `[0.0, 1.0]`（推力）内。

### 1.2 PX4 数据流向
1. **姿态环 (Attitude Control)**：计算出期望角速度 $\vec{\omega}_{sp}$ (单位：$rad/s$)。
2. **角速度环 (Rate Control)**：PID 输出归一化力矩指令 $\vec{\tau}_{norm} \in [-1.0, 1.0]^3$。
3. **位置环**：输出归一化推力指令 $T_{norm} \in [0.0, 1.0]$。
4. **控制分配 (Mixer)**：
   - 效率矩阵 $\mathbf{M}$ 也是基于归一化构建的，映射“控制指令比例”到“各个电机的油门比例 $u_i \in [0, 1]$”。
   - **去饱和逻辑 (Sequential Desaturation)**：如果算出某个电机油门 $u_i > 1.0$ 或 $< 0.0$，优先平移 $T_{norm}$，再缩放 $\vec{\tau}_{norm}$，输出合法的 $u_i \in [0.0, 1.0]$。
5. **底层驱动**：将 $u_i \in [0, 1]$ 映射到 PWM 或物理转速 $RPM$。

### 1.3 PX4 的优缺点
- **优点**：极度鲁棒。`[-1, 1]` 的阈值保护使得 PID 永远不可能输出物理上无法执行的“荒谬力矩”。参数通用性极强，换电机不需要重调 PID。
- **缺点**：丧失了直接的物理直觉。数学推导不如 SE3 那么优雅和严谨。

## 2. SE3 物理量纲架构 (Quadrotor_SE3_Control 案例分析)

`Quadrotor_SE3_Control` 是一个非常典型的基于几何控制（Geometric Control）的学术派实现。它的逻辑与我们目前的 `cascade_controller` 非常相似。

### 2.1 核心差异点对比 (与 PX4)

1. **误差计算在 SO(3) 流形上进行**
   - **PX4**：使用四元数或欧拉角，分离 Tilt (Roll/Pitch) 和 Yaw 误差，优先保证 Z 轴推力方向对齐。
   - **SE3**：使用旋转矩阵 $R$，通过 `vee` 映射直接计算 SO(3) 误差：
     $$e_R = \frac{1}{2} (R_d^T R - R^T R_d)^\vee$$
     这是一种全局无奇点的计算方式。
2. **纯物理量纲的 PID (几何控制器)**
   - SE3 的控制器直接输出**绝对物理量**：总推力 $f$ (单位 $N$) 和机体力矩 $M$ (单位 $Nm$)。
   - 它的增益（$k_R, k_\omega$）是与无人机的转动惯量 $J$ 强绑定的。
3. **Mixer 的直接物理逆解**
   - SE3 的 Mixer 矩阵 $\mathbf{M}$ 完全由物理参数（$C_T, C_D, L$）构成。
   - 输入是真实的 $N$ 和 $Nm$，输出是电机的真实转速平方（$kRPM^2$）。
4. **饱和处理 (Saturation Handling) 策略不同**
   - **PX4**：平移推力 -> 牺牲 Yaw -> 等比例缩放 Roll/Pitch。
   - **SE3 的 Mixer (参考其 `motor_mixer.py`)**：
     - 首先仅使用 $F_z, M_x, M_y$ 进行基础分配。
     - 检查 X/Y 轴的饱和情况。如果饱和，计算一个缩放因子 `scale` 强行按比例缩小 $M_x, M_y$。
     - 如果 X/Y 缩放后 `< 1.0`（说明电机能力已经用光了），直接丢弃 $M_z$（Yaw 扭矩），不予分配！
     - 只有在 X/Y 轴还有余量的情况下，才会将 $M_z$ 加上去，并再次检查 Z 轴是否导致饱和，如果是，则单独缩放 $M_z$。

### 2.2 SE3 Mixer 的精妙之处
在 SE3 项目的 Mixer 中，它并没有像 PX4 那样通过平移推力来保全姿态，而是采用了**严格的轴优先级丢弃法**：
1. **最高优先级**：推力 $F_z$（直接透传，不轻易平移）。
2. **第二优先级**：Roll / Pitch ($M_x, M_y$)。如果饱和，优先等比例缩小这两个。
3. **最低优先级**：Yaw ($M_z$)。如果前两者已经让电机饱和，Yaw 力矩直接置为 0。

## 3. 我们的项目该怎么选？

我们目前的架构（`cascade_controller.py`）实际上是 **SE3 控制器的变种**（在四元数上计算误差，输出物理 $N/Nm$）。

既然我们希望保持物理直觉，并且是在做算法仿真，**我们不需要（也不应该）完全重构为 PX4 的归一化架构**。相反，我们应该**吸收 SE3 控制器中处理物理饱和的精妙逻辑**。

### 我们的下一步行动方案（物理量纲下的稳定化）：

1. **坚持物理量纲**：继续使用 $N$ 和 $Nm$ 作为控制器输出和 Mixer 输入，这有助于我们理解无人机的真实动力学极限。
2. **重构 Mixer 饱和逻辑 (借鉴 SE3 源码)**：
   - 放弃我们之前写的 PX4 风格的“推力平移”。
   - 改为 SE3 风格的**分离计算与优先级截断**：
     - 阶段 1：先用 $[F_z, M_x, M_y, 0]^T$ 计算转速。
     - 阶段 2：如果超过 $[0, max\_speed^2]$，计算缩放比例 `scale` 压制 $M_x, M_y$。
     - 阶段 3：如果 `scale < 1.0`，直接返回（放弃 Yaw 控制）。
     - 阶段 4：如果还有余量，再加入 $M_z$ 重新计算。如果加入 $M_z$ 后饱和，单独计算 `scale_z` 压制 $M_z$。
3. **重新校准 PID 参数**：
   - 既然输出是 $Nm$，且最大可用扭矩只有 $0.0038\ Nm$（根据 SE3 代码），我们目前的 `RATE_KP = [0.2, 0.2, 0.2]` **绝对是太大了（大了一百倍以上）**！
   - 我们需要将 `RATE_KP` 降低到 `[0.001, 0.001, 0.001]` 级别，以匹配 $Nm$ 的数量级。
